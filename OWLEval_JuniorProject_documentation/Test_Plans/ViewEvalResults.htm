Introduction

 

The OwlEval system is designed to collect web-based course evaluations, and provide a secure interface for faculty to change evaluation questions and view reports of collected evaluations. The non-student option menu provides the choice of usage to the OwlEval system for OIT staff, administration, and faculty.

This test plan is the portion of the OwlEval system designed to link non-student personnel after entering a valid username and password to option available to the appropriate level of the system.

 

Relationship to Other Documents

 

The OwlEval system is compose of five general components:

 

1.      Student evaluation completion

2.      Non-student login, menu, and reports viewing

3.      Evaluation augmentation

4.      Evaluation creation and key generation

5.      Database access

            Since each page must complete processing before it is displayed to the user (because it will be written in PHP), any error condition must be noted and displayed as part of the normal page processing routine. Suspension and resumption will generally be handled in this way.
