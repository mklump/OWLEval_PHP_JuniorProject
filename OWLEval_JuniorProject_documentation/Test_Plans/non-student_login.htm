Introduction

 

The OwlEval system is designed to collect web-based course evaluations, and provide a secure interface for faculty to change evaluation questions and view reports of collected evaluations. The non-student login page provides access security to the OwlEval system for OIT staff, administration, and faculty.

This test plan is a portion of the OwlEval system designed to have non-student personnel enter a username and password. After validation of the username and password a menu of available option will be presented

 

Relationship to Other Documents

 

The OwlEval system is compose of five general components:

 

1.      Student evaluation completion

2.      Non-student login, menu, and reports viewing

3.      Evaluation augmentation

4.      Evaluation creation and key generation

5.      Database access

Approach

 

            To test this Page, the basic page framework and external variables are assumed proper and without error.

 

Suspension and resumption 

            Since each page must complete processing before it is displayed to the user (because it will be written in PHP), any error condition must be noted and displayed as part of the normal page processing routine. Suspension and resumption will generally be handled in this way.
