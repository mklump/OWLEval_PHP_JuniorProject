Student Module Test Plan

Introduction

The OwlEval  system is designed to collect web-based course evaluations, and provide a secure interface for faculty to change evaluation questions and view reports of collected evaluations. The student portion of the OwlEval system is where the evaluations are actually collected

This portion of the OwlEval system is designed to have a student enter a key, be presented with an evaluation corresponding to that key, fill out the evaluation, and save it. It must handle erroneous data entry, and entry onto a page outside of the standard flow.

 

Relationship to Other Documents

 

            We have broken the OwlEval system into five components:

 

1.      Student evaluation completion

2.      Faculty login and reports viewing

3.      Evaluation augmentation

4.      Evaluation creation and key generation

5.      Database access

 

This document focuses on item 1. Accordingly, there will also be test cases for each of the other items.

Approach

 

            To test these features, the basic page framework will be constructed with external variables assumed and proper. Each feature will be constructed and verified with all feasible inputs in an attempt to crash the system.

Suspension and resumption 

            Since each page must complete processing before it is displayed to the user (because it will be written in PHP), any error condition must be noted and displayed as part of the normal page processing routine. Suspension and resumption will generally be handled in this way.

Testing materials (hardware/software requirements)

            One PC with PHP and MySQL installed locally.
