Project Deliverable 5
  

Project Deliverable 5.1 - Test Plan

 

Instructor modify and save questions test plan:

1. Introduction

     This will test the web pages that the instructor uses to modify and save evaluation questions, which the instructor wishes to use for their course evaluations.

 

2. Relationship to other documents

     The web page will be loaded from the non-student menu web page.

 

3. System overview

     From the non-student menu page the Instructor clicks on the hyperlink to add to or modify their evaluation questions. The first page to load will show a list of the questions that the instructor has made up in previous sessions. If no questions have been made up then a message will be displayed to that effect. The list of questions will also have the CRNs of which classes the questions apply to. There will be a button on the page which can be clicked on to add a new question to their list. The questions will be hyperlinked which can be clicked on so the question can be modified and to apply the question to a course CRN. The questions page will have a link to other instructor's questions that the instructor can choose to use. If an instructor wishes to use another instructor's question then the question will be duplicated and stored as one of his/her questions. This way any further modifications will only occur to the question that the instructor owns and won't affect the other instructors questions. When a question is clicked on another web page will be loaded with the question in a text box so it can be edited and the questions attributes will be displayed such as the CRNs that the question is associated with. The questions cannot be modified when the evaluation is available to the students.

 

4. Features to be tested/not to be tested

     a. Ensure that the questions that the instructor has created or adopted are displayed in their view.

     b. Ensure that other instructor's questions do not appear in their view.

     c. Ensure that only the instructor can modify questions that the instructor adopts from other instructors.

     d. Ensure that questions that the instructor modifies do not affect any other questions.

     e. Ensure that only the individual instructor can be able to access their questions.

     f.  Ensure that only the individual instructor can be able to assign their questions to only their courses.

     g. Ensure that the questions cannot be modified when the evaluation is available to the students.

     h. Ensure that the questions in the instructors view are correctly associated with CRNs.

     i.  Ensure that all the other instructor's questions are loaded when the hyperlink to other instructor's questions is clicked.

 

5. Pass/Fail criteria

     a. Fail if any of the instructor's questions is not displayed in their view.

     b. Fail if any questions that are not the instructor's are displayed in their view.

     c. Fail if any of the questions that the instructor adopts from other instructors cannot be modified by them or can be modified by someone else.

     d. Fail if any of the questions that the instructor modifies affect any other questions.

     e. Fail if any instructor can access other instructor's questions.

     f.  Fail if any instructor can assign their questions to any other instructor's courses.

     g. Fail if the questions can be modified when the evaluation is available to the students.

     h. Fail if the questions in the instructors view are not correctly associated with CRNs or do not appear when they are supposed to show up.

     i.  Fail if any of the other instructor's questions are not loaded when the hyperlink to other instructor's questions is clicked.

 

6. Approach

     At least 2 people who are not familiar with the project design or development will use a client PC to access the evaluation web site, log in, and perform the various aspects of the evaluation system. Two people will be the instructors to create, modify, and adopt questions for their evaluations. Attempt to modify questions when the evaluation is available to the students. The design and development team will not interact with the users during the test but will make notes of the problems observed.

 

7. Suspension and resumption 

     The two people will be the instructors will alternate tasks so that they can adopt questions from each other, modify their question, and check their questions to make sure that they aren't affected by the other instructors. Suspend when the evaluation is being made available to the students. Resume when the evaluation is available to the students. Suspend when the evaluation is being closed. Resume when the evaluation has been closed.

 

8. Testing materials (hardware/software requirements)

     a. Client PC

          1) Internet/Ethernet connection

          2) Internet Explorer or other web browser

     b. Web server (such as Apache on Linux or IIS on Windows)

          1) Internet/Ethernet connection

          2) PHP installed

          3) Database (such as MySQL)

 

9. Test cases

     Prerequisites: The instructor's and their CRNs must be in the database.

     a. Instructor one: Enter the add to or modify evaluation questions section.

          1) From the non-student menu page the instructor clicks on the hyperlink to add to or modify their evaluation questions.

          2) A web page with the instructor's list of questions with CRNs (if assigned) is loaded or a message is displayed if there are no questions.

          3) Fail if any of the instructor's questions are not displayed in their view.

          4) Fail if any questions that are not the instructor's are displayed in their view.

          5) Fail if any page is not rendered properly.

     b. Instructor one: Add a question.

          1) From the instructor's list of questions the instructor clicks on the button to add a question.

          2) A web page will be loaded with an empty text box where the new question will be typed in and the instructor's course CRNs that the question can be associated with.

          3) Type the new question into the text box. Select a check box next to each CRN to associate the question with that CRN.

          4) Click on the save button.

          5) The previous page with the list of questions with CRNs (if assigned) is loaded.

          6) Fail if the new question is not displayed in the instructor's list of questions or the CRNs are not associated properly.

          7) Fail if any page is not rendered properly.

     c. Instructor one: Modify a question.

          1) From the instructor's list of questions the instructor clicks on the question that the instructor wishes to modify or associate with CRNs.

          2) A web page will be loaded with the question in a text box and the instructor's course CRNs that the question is associated with and the remaining CRNs that the question can be associated with.

          3) Edit the question in the text box. Select a check box next to each CRN to associate the question with that CRN.

          4) Click on the save button.

          5) The previous page with the list of questions with CRNs (if assigned) is loaded.

          6) Fail if the modified question is not displayed in the instructor's list of questions or the CRNs are not associated properly.

          7) Fail if any page is not rendered properly.

     d. Instructor two: Perform test case a., b., and c. above.

     e. Instructor two: Adopt a question from instructor one.

          1) From the instructor's list of questions the instructor clicks on the button to adopt a question from another instructor.

          2) A web page will be loaded with a list of all other instructor's questions.

          3) Select a check box next to each question to adopt.

          4) Click on the save button.

          5) The previous page with the list of questions with CRNs (if assigned) is loaded.

          6) Fail if the adopted question is not displayed in the instructor's list of questions or the CRNs are not associated properly.

          7) Fail if any page is not rendered properly.

     f. Repeat above as needed.
