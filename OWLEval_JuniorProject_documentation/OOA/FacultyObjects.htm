Faculty member logs into program at the  admin/faculty login interface with login/password. The admin/faculty login interface checks the user database and determines that the login/password are valid and that the user is a faculty member to be sent to the primary faculty interface.

 

Faculty member is transferred to primary faculty interface, showing them all of the evaluations in the system to which they are associated, culled from the evaluation database.

 

Faculty member goes to personal information interface and may change password and contact information. Submitting changes to the personal contact information and password, the changes are verified and the user database is updated with the changes. The faculty member is transferred back to the primary faculty interface.

 

From the primary faculty interface, selecting one or many  completed/in-progress evaluations, the faculty member is transferred to a results interface and is shown the results of the evaluation(s) culled from the evaluation database. When the faculty member is finished examining the the results of the evaluation(s), the faculty member clicks a link and is transferred back to the primary faculty interface.

 

Faculty member selects one or many unstarted evaluations and is transferred to the instructor augmentations interface. In the instructor augmentations interface, the faculty member types in essay and rated questions. Submitting the instructor augmentations  stores them in the evaluation database pending approval by the administrators. The faculty member is transferred back to the primary faculty interface..

 

The faculty member may logout of the system at any time.

 

Nouns

Faculty Member – Actor

Program – The entire OwlEval evaluation system

Admin/Faculty Login Interface – Object that allows Administrators and Faculty members to log into the back end of the system

Login/Password – Data Object stored in user database

User Database – Data Store containing Password and Contact Information for administrators and faculty members.

User – Actor, either a Student, an Administrator, or a Faculty Member

Primary Faculty Interface – Object that presents a list of evaluations that are associated with a particular Faculty Member.

Evaluation – Data Object that contains a number of student keys, a course(by CRN), one or many Faculty Members, any approved or unapproved instructor augmentations, and results of the evaluation along with the results for approved instructor augmentations.

System – the Entire OwlEval Evaluation System

Personal Information Interface – Object that displays an interface allowing users to edit their personal contact information and password.

Password and Contact information – Data Object stored in user database. Contains Login/Password as element.

Changes- Data Object consisting of the modifications that the user has made to the pertinent data. Changes are verified.

Completed/In-Progress Evaluations – The set of Evaluation Objects that include Evaluations that have already been completed or are currently in-progress.

Evaluation Database – Data Store where Evaluation Objects are stored.

Results Interface – Object that displays the results of one or many Evaluations

Results of the Evaluations – the results of the evaluations, stored in an evaluation object

Link – Button that performs an interface switching operation without saving anything.

Unstarted Evaluations – The set of evaluations that have yet to begin, but are already entered into the Evaluation Database.

Instructor Augmentation Interface – Object that presents the instructor with the opportunity to enter new essay questions and rated questions.

Essay and Rated Questions –Data Objects that represent either an Essay Question or a Scale Rated question.
