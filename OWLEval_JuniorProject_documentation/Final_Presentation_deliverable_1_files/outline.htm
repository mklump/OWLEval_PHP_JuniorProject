Project Overview
Problem definition
The current paper based system is too labor intensive and does not provide feedback.

Project goal statement
Provide a portable, web-based, secure evaluation system by the end of Spring term 2004.
Project Sponsors
Abbie Allen (OIT Metro)
Janelle Knaggs (OIT Klamath Falls)
Cheryl Jenkins (OIT Capital Center)
Project Requirements
1) Compatible with OIT’s current hardware, software, and process.
1.1) Shall be designed in such a way that connecting to OIT’s system will be possible in the future.
Problem: It may not be implemented even if it works well and is compatible.
1.2) Shall reduce paperwork for office staff.
1.3) Shall reduce time invested by office staff.
Problem: If it is not implemented then these won’t be achieved.
Project Requirements
2) Web Based Interface
2.1) Shall work correctly with at least one web browser available at OIT.
2.2) Shall separate interface based on user role.
2.3) Shall provide a help/feedback function.
Problem: We have not designed this function.
Project Requirements
3) Secure
3.1) Shall Identify students only with randomly generated access key and password.
3.2) Shall restrict data input to access keys identified with a particular course.
3.3) Shall secure communications with Web Clients.
3.4) Shall restrict access to database.
3.5) Shall restrict access to program based on Role.
3.6) Shall allow faculty to change their own passwords.
3.7) Shall allow administrators to change any password.
3.8) Shall be deployed only within OIT facilities.
Project Requirements
4) Collects, Displays, and Processes Evaluation Data
4.1) Data shall be delivered according to current standards.
4.2) Shall store Course information and data collected from students.
4.3) Shall not lose data.
4.4) Shall generate reports for a course either individually or as a batch process.
4.5) Shall allow students to save a partially completed evaluation so that they may continue at a later time.
Change: It will be a one time only evaluation.
4.6) Shall allow administrator to archive and remove data from database
Project Requirements
5) User-Configurable
5.1) Shall allow administrators to create, modify and delete static accounts for faculty as they see fit.
5.2) Shall allow administrators to create, modify, delete, and reorder evaluation form questions.
Change: Instructors will modify their own questions.
Project Requirements
6) Maintenance Free
6.1) After initial installation, there shall be no further system level access required.
6.2) After installation, all necessary functions shall be performed from the user interface.
Project Requirements
7) System has Role-based Availability
7.1) Administrators shall have access to system at all times.
7.2) Faculty may see results at any time.
Change: Only after the evaluation is closed out by the administrator.
7.3) Students shall be restricted to a certain time period for entering data
Project Requirements
8) Testing
8.1) Shall be able to handle multiple simultaneous submissions.
8.2) Shall accept all printable characters.
8.3) Shall work correctly
