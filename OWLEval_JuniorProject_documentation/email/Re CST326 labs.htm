For a broad concept as to how this is going to work, think of how an
e-commerce store works. You go to a web page and are presented with a log
in screen that's written in HTML. The HTML may have been written via ASP,
or JSP or PHP, but in the end it's just an HTML web page. You interact
with the page, submitting your information using a simple HTML form. This
information (along with some hidden information provided without your
knowledge by your browser) is sent to a script that runs on the server.
That script takes the information you've submitted and manipulates
databases, or files, or whatever the programmers have decided it should
manipulate, and then dynamically generates and sends an appropriate HTML
response to your browser.

This is the essential interaction that defines how our project will
function. The client points his browser at a web page, our program sends
the appropriate web page in response, the user interacts with the web page
and submits it back to a script on our server which generates another
appropriate response.

This is interaction is an archetype for most dynamic content web pages and
web applications that exist on the web today.
