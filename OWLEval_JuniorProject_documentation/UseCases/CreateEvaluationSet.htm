Use Case: Create evaluation set (tables)

 

Description:  This Use Case describes the steps a member of administration takes in the process of making the individual evaluation packages for each course (CRN).

 

Actor(s): Staff

 

Preconditions: Specific course (CRN) must be under consideration for evaluation.

 

Basic Path: 

This is an existing computer process of gathering information for a particular CRN   from the Student Information System (SIS).

This Use Case is initiated when:

 

  1. Staff logs in to SIS
  2. Data for each course (CRN) selected is downloaded
  3. A table is built to receive CRN specific information
  4. Course evaluation set (tables) are stored as “Boeing” set or regular set

 

Alternate Paths:

 

2.0)          Certain Courses (CRN) may be excluded such as independent study, seminars, or distance education

2.1)           Return to basic path at step 3

 

 

Exception Paths:

No known user exception paths. System to validate user and information

 

Post conditions:

Completed evaluation set (tables) for all specified courses, CRN’s

 

Related Data:

Merged evaluation sets (tables),

 

Business Rules: 

No known constraints at this level

 

Requirements:

This Process is begun the Monday following the “last day to drop a course with a Withdrawal”
