Use Case:  Staffing Editing

 

Description:  This Use Case describes the process in which Staffing edits the recent course evaluations to the desired format to store data. This Use case is completed after the Staff member commits course evaluation (CE).

 

Actor(s):  Staffing (member of)

 

Preconditions:  School term is concluding and course evaluation (CE) has been submitted

 

Basic Path:  This Use Case is initiated when:

1.     Staffing accesses the system using the courses CRN

2.     The System then generates current CE’s for that CRN

3.     Staff will proof read CE and edit

4.     Staff formats input to DBMS

5.     Staff commits CE to storage

6.     Staffing receives confirmation that the newly edited CE was added successfully.

 

Alternate Paths:

5.0) CE is stored in temp file awaiting completion

5.1) Necessary information is obtained

5.2) Return to step 3

 

Exception Paths:

2.0) System generates CE for different CRN

2.1) Staff invokes Use Case to determine if activity key is valid for the CRN

2.2) Staff receives report Activity key validation

2.3) Invoke appropriate Error handler

2.4) Return to Basic Path step 3

 

Post conditions:

CE status is confirmed and ready for report generation as catagorized

 

Related Data:

All Student submitted CE’s relevant to particular CRN

 

Related Data:

Student Information System (SIS)

 

Business Rules: 

The actual filling out of forms happens the last two weeks before finals week.

 

Requirements:

Student must be currently enrolled in course before access key is valid (issued)
